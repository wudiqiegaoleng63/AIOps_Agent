## C 关键设计决策清单

| # | 决策 | 原因 |
|---|------|------|
| 1 | `past_steps` 用 `Annotated[..., operator.add]` | 追加语义，执行历史只增不减 |
| 2 | MCP 服务器独立进程 | 工具服务独立部署升级，不影响主服务 |
| 3 | MCP 客户端单例 + 重试拦截器 | 避免重复连接，工具调用失败自动重试 |
| 4 | Agent 延迟初始化 | MCP 工具需要异步获取，启动时不必就绪 |
| 5 | Executor 固定三阶段协议 | 防止无限工具调用循环 |
| 6 | Replanner 三优先级（respond > continue > replan） | 偏向提前结束，防止过度执行 |
| 7 | 多层硬限制（8步上限/5步禁replan/新步骤数限制） | 兜底防无限循环 |
| 8 | Replanner 通过状态变更间接表达决策 | 条件边函数不依赖 Act 模型，更解耦 |
| 9 | 文档分块 chunk_size 加倍（1600 非配置值 800） | 减少碎片，提高检索质量 |
| 10 | Milvus monkey-patch 统一连接别名 | 修复 langchain_milvus 与 pymilvus ORM 的兼容性问题 |
| 11 | 先删后加的文档更新策略 | 支持文档覆盖上传 |
