## 4.2 文档分割（3 阶段 Markdown 分割）

这是文档处理的核心，分 3 阶段处理 Markdown：

```python
# app/services/document_splitter_service.py
class DocumentSplitterService:
    def __init__(self):
        # 阶段1: 按标题分割
        self.markdown_splitter = MarkdownHeaderTextSplitter(
            headers_to_split_on=[("#", "h1"), ("##", "h2")],
            strip_headers=False,  # 保留标题在内容中
        )
        # 阶段2: 按大小再分割
        self.text_splitter = RecursiveCharacterTextSplitter(
            chunk_size=config.chunk_max_size * 2,  # 加倍减少碎片
            chunk_overlap=config.chunk_overlap,
        )

    def split_markdown(self, content: str, file_path: str = "") -> List[Document]:
        # 阶段1: 按 h1/h2 标题分割
        md_docs = self.markdown_splitter.split_text(content)
        # 阶段2: 大块再分割
        docs_after_split = self.text_splitter.split_documents(md_docs)
        # 阶段3: 合并太小的分片 (< 300 字符)
        final_docs = self._merge_small_chunks(docs_after_split, min_size=300)
        # 添加元数据
        for doc in final_docs:
            doc.metadata["_source"] = file_path
            doc.metadata["_extension"] = ".md"
            doc.metadata["_file_name"] = Path(file_path).name
        return final_docs
```

**3 阶段分割原理**：

```
原始 Markdown
    │
    ▼ 阶段1: MarkdownHeaderTextSplitter（按 # 和 ## 分割）
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 大块 A    │ │ 大块 B    │ │ 大块 C    │
└──────────┘ └──────────┘ └──────────┘
    │
    ▼ 阶段2: RecursiveCharacterTextSplitter（按大小再分割）
┌────┐┌────┐ ┌──────────┐ ┌──┐┌──────┐
│ A1 ││ A2 │ │ B(合适)   │ │C1││ C2   │  C1 太小
└────┘└────┘ └──────────┘ └──┘└──────┘
    │
    ▼ 阶段3: 合并小碎片 (< 300 字符)
┌────┐┌────┐ ┌──────────┐ ┌─────────┐
│ A1 ││ A2 │ │ B        │ │ C1+C2   │  ← C1 和 C2 合并
└────┘└────┘ └──────────┘ └─────────┘
```

**要点**：
- `chunk_size` 用 `config.chunk_max_size * 2`（1600 而非 800），是为了减少碎片数量
- 阶段 3 合并小碎片很重要，太短的片段检索效果差
- 纯文本文件只走阶段 2，直接 `RecursiveCharacterTextSplitter`

## 4.3 索引完整流程

```python
# app/services/vector_index_service.py
class VectorIndexService:
    def index_single_file(self, file_path: str):
        # 1. 读取文件
        content = Path(file_path).read_text(encoding="utf-8")
        # 2. 删除该文件的旧数据（支持重复上传）
        vector_store_manager.delete_by_source(normalized_path)
        # 3. 分割文档
        documents = document_splitter_service.split_document(content, normalized_path)
        # 4. 添加到向量存储（自动 embedding + 入库）
        if documents:
            vector_store_manager.add_documents(documents)
```

**要点**：先删后加，支持文档覆盖更新。

## 4.4 知识检索工具

供 Agent 调用的 LangChain Tool：

```python
# app/tools/knowledge_tool.py
@tool(response_format="content_and_artifact")
def retrieve_knowledge(query: str) -> Tuple[str, List[Document]]:
    """从知识库中检索相关信息"""
    vector_store = vector_store_manager.get_vector_store()
    retriever = vector_store.as_retriever(search_kwargs={"k": config.rag_top_k})
    docs = retriever.invoke(query)

    if not docs:
        return "没有找到相关信息。", []

    return format_docs(docs), docs  # 返回 (格式化文本, 原始文档)


def format_docs(docs: List[Document]) -> str:
    """格式化文档为上下文文本"""
    parts = []
    for i, doc in enumerate(docs, 1):
        source = doc.metadata.get("_file_name", "未知来源")
        headers = [doc.metadata[k] for k in ["h1", "h2", "h3"] if k in doc.metadata]
        formatted = f"【参考资料 {i}】\n来源: {source}"
        if headers:
            formatted += f"\n标题: {' > '.join(headers)}"
        formatted += f"\n内容:\n{doc.page_content}"
        parts.append(formatted)
    return "\n".join(parts)
```

**要点**：
- `response_format="content_and_artifact"` 让工具同时返回格式化文本和原始文档
- `format_docs` 保留标题层级信息，方便 Agent 理解文档结构
